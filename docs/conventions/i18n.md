# Мова — українська (єдина)

> Проєкт використовує тільки українську мову. Мультимовність видалено.

## Принцип

```
Frontend = українська (hardcoded)
Backend  = англійська (code + message для розробників)
Emails   = українська (hardcoded)
```

## Backend: тільки англійська

### API responses

Бекенд **НІКОЛИ** не повертає user-facing тексти. Відповідь складається з двох частин:

1. **code** — машиночитаний код для фронтенду (маппиться на українське повідомлення)
2. **message** — коротке пояснення англійською для розробника (логи, дебаг)

#### Success response

```typescript
return {
    data: {
        code: 'MAGIC_LINK_SENT',
        message: 'Magic link sent',
    },
};
```

#### Error response (через AllExceptionsFilter)

```json
{
    "error": {
        "code": "RATE_LIMIT_EXCEEDED",
        "message": "Too many requests. Try again in 15 minutes."
    }
}
```

### Що заборонено на бекенді

- Повертати user-facing повідомлення (тости, нотифікації)
- Хардкодити локалізований текст у responses

### Emails

Email templates написані українською. Мова зашита в шаблони напряму.

## Frontend: українська

### Маппинг кодів на повідомлення

Фронтенд отримує `code` з API та маппить на українське повідомлення через `getApiMessage()` (`shared/api/mapApiCode.ts`).

```
API code: "MAGIC_LINK_SENT"
  → getApiMessage('MAGIC_LINK_SENT')
  → "Посилання надіслано на вашу пошту"
```

### Тексти у компонентах

Всі user-facing тексти інлайнені безпосередньо в компоненти як строкові літерали. Бібліотеки i18n не використовуються.

**НІКОЛИ** не показувати `message` з API користувачу — це текст для розробника.

## Response Code Registry

Всі коди зареєстровані в enum `RESPONSE_CODE` (`packages/types/src/enums/response-code.ts`).

### Як додати новий код

1. Додай рядок в `RESPONSE_CODE`
2. Додай маппінг в `RESPONSE_CODE_TYPE` (код → тип)
3. Додай маппінг в `MESSAGES` об'єкт в `shared/api/mapApiCode.ts`
