# FinFlow — Product Vision

## 1. Суть продукту (Core Vision)

FinFlow — це мікро-SaaS для створення динамічних платіжних посилань та брендованих QR-кодів за стандартом НБУ. Продукт вирішує єдину проблему: усуває тертя при оплаті на рахунки ФОП. Замість ручного копіювання 29 символів IBAN, платник в один клік відкриває свій банківський застосунок із вже заповненими реквізитами.

## 2. Головна проблема ринку та рішення (Кіллер-фіча)

**Проблема:** Базові посилання НБУ (bank.gov.ua/qr/...) блокуються або некоректно працюють у вбудованих браузерах Instagram, TikTok та Facebook (In-App Webview).

**Рішення (Smart Redirect):** FinFlow генерує не пряме посилання, а лінк на власну сторінку-прокладку (Payment Gateway, напр. pay.finflow.ua/shop-name). На цій сторінці платник бачить суму та кнопки: "Оплатити через Monobank", "Приват24", "Sense Bank". Клік по кнопці ініціює нативний системний intent, який гарантовано пробиває обмеження соцмереж і відкриває потрібний банк.

## 3. Цільова аудиторія

**Primary (Платний сегмент):** Бухгалтери на аутсорсі та консалтингові агенції. Вони купують платформу для зручного управління десятками своїх клієнтів (ФОПів) з одного кабінету.

**Secondary (Безкоштовний сегмент):** Мікробізнес та Instagram-магазини. Вони використовують безкоштовний тариф, виступаючи носіями вірусної реклами платформи.

## 4. Бізнес-модель (Product-Led Growth)

Платформа монетизується за моделлю Freemium із жорстким Paywall для B2B-функцій.

### Free Tier (Для генерації трафіку)

- **Ліміт:** Строго 1 ФОП на акаунт.
- Базовий Smart Redirect та QR-код.
- **Умова:** На сторінці оплати невідключуваний водяний знак (Watermark): "Рахунок виставлено через FinFlow. Створити свій безкоштовно." (Вірусний цикл залучення).

### Pro / Agency Tier (Основна монетизація)

- **Мультиакаунти:** Можливість додати від 5 до 100+ ФОПів у єдиний дашборд. (Критично для бухгалтерів).
- **White-Label:** Завантаження логотипа клієнта на сторінку оплати та видалення водяного знака FinFlow.
- **Динамічні лінки:** Можливість змінити суму або призначення платежу в адмінці без зміни самого URL-посилання (зручно для постійних клієнтів).
- **Аналітика:** Відстеження кількості кліків по кожному посиланню.

## 5. Технічна архітектура та безпека

- **Stateless Платежі:** FinFlow не бере участі в транзакціях і не знає про їхній статус. Усі платіжні дані інкапсульовані в Base64URL. Це знімає будь-які юридичні ризики та необхідність сертифікації PCI DSS.
- **Стек:** Next.js (фронтенд та сторінки редіректів), NestJS (API та логіка мультиакаунтів), PostgreSQL + Prisma (строга реляційна база для зв'язків Бухгалтер -> ФОПи -> Лінки).
- **Відсутність еквайрингу:** Платформа позиціонується як легальна альтернатива еквайрингу (економія 1.5-3% на комісіях для мікробізнесу). Сама підписка на FinFlow оплачується через стандартний інтернет-еквайринг.
