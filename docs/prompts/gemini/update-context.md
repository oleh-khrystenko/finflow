## Інструкції

### Крок 1 — Глибокий Аналіз
Рекурсивно проскануй ВСЮ кодову базу. Враховуючи велике контекстне вікно, проаналізуй не тільки структуру, а й неочевидні зв'язки:
- **Архітектура:** Як компоненти взаємодіють (Flow даних).
- **Domain Model:** Сутності, БД (схема), типи та DTO.
- **Dependency Graph:** Хто від кого залежить (imports analysis).
- **Patterns:** Повторювані рішення (Authentication, Error Handling, Validation pipelines).
- **Config & Environment:** Які змінні керують поведінкою системи.
- **Testing Strategy:** Покриття та підходи (E2E, Unit, Integration).

### Крок 2 — Збереження Manual Knowledge
Прочитай поточний `GEMINI.md` (якщо існує). Весь контент між маркерами `` та `` перенеси в новий файл **БЕЗ ЗМІН** — дослівно. Це зона відповідальності людини (бізнес-контекст, специфічні домовленості).

### Крок 3 — Генерація Контексту
Перепиши `GEMINI.md`, використовуючи наведену нижче структуру. Пиши **стисло, технічно, фактами**. Це файл для твоїх наступних сесій (пам'ять агента), а не документація для людей.
