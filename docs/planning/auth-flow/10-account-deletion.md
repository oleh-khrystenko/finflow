# Видалення акаунту

## Де знаходиться

Сторінка профілю → секція **"Небезпечна зона"** в самому низу сторінки, візуально відокремлена червоною рамкою. Всередині — заголовок "Видалення акаунту", короткий опис "Після видалення у вас є 30 днів для відновлення акаунту" та червона кнопка "Видалити акаунт".

## Що означає видалення

Видалення акаунту — це не миттєве знищення даних. Акаунт **деактивується** і переходить у стан "очікує видалення". Протягом наступних **30 днів** користувач може передумати та відновити акаунт. Якщо за 30 днів цього не сталося — акаунт і всі пов'язані дані видаляються повністю і безповоротно.

---

## Шлях А — Акаунт із паролем

Якщо користувач встановив пароль для свого акаунту, підтвердження видалення відбувається через пароль.

**Крок 1.** Користувач натискає червону кнопку **"Видалити акаунт"**.

**Крок 2.** Поверх сторінки з'являється модальне вікно:

```
┌─────────────────────────────────────────┐
│         Видалення акаунту               │
│                                         │
│  Ви впевнені? Ваш акаунт буде           │
│  деактивовано. У вас буде 30 днів       │
│  для відновлення.                       │
│                                         │
│  Введіть пароль для підтвердження       │
│  ┌───────────────────────────────────┐  │
│  │  ••••••••                         │  │
│  └───────────────────────────────────┘  │
│                                         │
│  [ Скасувати ]  [ Видалити акаунт ]     │
└─────────────────────────────────────────┘
```

**Крок 3.** Користувач вводить свій пароль і натискає **"Видалити акаунт"**.

- Якщо пароль **невірний** — під полем з'являється помилка "Невірний пароль". Модальне вікно залишається відкритим, можна спробувати ще раз або натиснути "Скасувати".
- Якщо пароль **вірний** — модальне вікно закривається.

**Крок 4.** Акаунт деактивується, всі активні сесії на всіх пристроях завершуються. На пошту надходить лист із підтвердженням видалення та датою остаточного знищення даних.

**Крок 5.** Сторінка автоматично переходить на сторінку входу. Більше зайти в акаунт звичайним шляхом не вдасться — але є можливість відновити (детальніше нижче).

---

## Шлях Б — Акаунт без пароля (тільки Google або magic link)

Якщо користувач входить виключно через Google або email-посилання і пароля не встановлював, підтвердження видалення відбувається через email-посилання.

**Крок 1.** Користувач натискає червону кнопку **"Видалити акаунт"**.

**Крок 2.** Сторінка не переходить нікуди. Прямо під описом "Після видалення у вас є 30 днів для відновлення акаунту" з'являється синій інформаційний блок:

```
┌─────────────────────────────────────────────────────────┐
│  Небезпечна зона                                        │
│                                                         │
│  ┌─────────────────────────────────────────────────┐    │
│  │  Видалення акаунту                              │    │
│  │  Після видалення у вас є 30 днів для            │    │
│  │  відновлення акаунту.                           │    │
│  │                                                 │    │
│  │  ┌──────────────────────────────────────────┐  │    │
│  │  │  Перевірте пошту                         │  │    │
│  │  │                                          │  │    │
│  │  │  Посилання для підтвердження видалення   │  │    │
│  │  │  акаунту надіслано на вашу адресу.       │  │    │
│  │  │  Посилання дійсне 15 хвилин.             │  │    │
│  │  └──────────────────────────────────────────┘  │    │
│  │                                                 │    │
│  │  [ Надіслати повторно (60с) ]                   │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

Одночасно на пошту надходить лист із посиланням для підтвердження видалення. Посилання дійсне **15 хвилин**.

**Крок 3 — Кнопка "Надіслати повторно".** Після відправки кнопка на 60 секунд стає неактивною з відліком "Надіслати повторно (59с)", "Надіслати повторно (58с)" і т. д. Після закінчення відліку кнопка знову стає активною. Це потрібно щоб уникнути випадкового подвійного натискання та захистити від спаму.

**Крок 3а — Якщо користувач перезавантажив сторінку.** Синій інформаційний блок і кнопка "Надіслати повторно" залишаються видимими — стан зберігається до тих пір, поки посилання ще дійсне (15 хвилин від моменту останнього запиту). Після закінчення 15 хвилин сторінка повертається до початкового стану з кнопкою "Видалити акаунт" — можна починати процес заново.

**Крок 3б — Якщо натиснути "Надіслати повторно" занадто багато разів.** Система дозволяє не більше 3 запитів на відправку посилання за 15 хвилин. При перевищенні ліміту з'являється повідомлення "Забагато запитів. Спробуйте через 15 хвилин".

**Крок 4.** Користувач відкриває лист і натискає посилання. Відкривається сторінка підтвердження:

```
┌─────────────────────────────────────────┐
│                                         │
│            ✅                           │
│                                         │
│         Акаунт видалено                 │
│                                         │
│  Ваш акаунт деактивовано. Протягом      │
│  30 днів ви можете відновити його —     │
│  просто увійдіть до системи.            │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │             Увійти                │  │
│  └───────────────────────────────────┘  │
│                                         │
└─────────────────────────────────────────┘
```

Акаунт деактивовано, всі активні сесії на всіх пристроях завершені. На пошту одночасно надходить лист із підтвердженням видалення та датою остаточного знищення даних.

**Крок 4а — Якщо посилання вже прострочене або використане.** Замість екрану підтвердження з'являється повідомлення "Посилання недійсне або прострочене" з кнопкою "Спробувати знову" (веде на сторінку входу).

## Confirmation email (обидва шляхи)

Після деактивації акаунту на пошту надходить лист з:
- підтвердженням що акаунт деактивовано
- датою остаточного видалення (поточна дата + 30 днів)
- інструкцією як відновити: увійти в систему протягом 30 днів

---

## Grace period — 30 днів для відновлення

Після деактивації акаунт існує в системі ще 30 днів. У цей час:

- Зайти на платформу та користуватися нею **неможливо**
- **Але** увійти через форму авторизації (пароль / email-посилання / Google) можна — однак замість звичайного переходу в профіль з'явиться спеціальний екран відновлення

## Екран відновлення

Коли користувач з деактивованим акаунтом успішно входить (вводить правильний пароль або переходить по magic link), замість профілю він бачить:

```
┌─────────────────────────────────────────┐
│                                         │
│         Акаунт деактивовано             │
│                                         │
│  Ваш акаунт було видалено 04.03.2026.   │
│  Він буде остаточно видалено через      │
│  27 днів.                               │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │         Відновити акаунт          │  │
│  └───────────────────────────────────┘  │
│                                         │
│              Вийти                      │
│                                         │
└─────────────────────────────────────────┘
```

- **"Відновити акаунт"** — акаунт повністю відновлюється, всі дані збережені, користувач переходить у профіль і може користуватися платформою як раніше.
- **"Вийти"** — користувач виходить із системи і потрапляє на сторінку входу. Акаунт залишається деактивованим.

---

## Остаточне видалення після 30 днів

Щодня о 3:00 ночі система автоматично перевіряє, чи є акаунти, у яких минуло 30 днів з моменту деактивації. Такі акаунти видаляються повністю і безповоротно: профіль, дані, платежі, кредити — все. Після цього email звільняється і може бути використаний для нової реєстрації.

## Magic link purposes

| Purpose | Сценарій | Результат після кліку |
|---------|----------|---------------------|
| `login` | Існуючий юзер без пароля | Вхід → профіль (пароль — редагування) |
| `register` | Новий юзер | Вхід → профіль (повне редагування) |
| `reset-password` | Забув пароль | Вхід → профіль (пароль — обов'язкове редагування) |
| `delete-account` | Підтвердження видалення | Деактивація → екран підтвердження → кнопка "Увійти" |

## User schema

```
deletedAt: Date | null               // null = активний, Date = деактивований
accountDeletionRequestedAt: Date | null  // null = не запитував, Date = чекає підтвердження email
```
