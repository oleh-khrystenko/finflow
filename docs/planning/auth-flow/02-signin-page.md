# Сторінка авторизації

## UI елементи

```
┌─────────────────────────────────────────┐
│                                         │
│            Вхід до LucidShip             │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │      Увійти з Google              │  │
│  └───────────────────────────────────┘  │
│                                         │
│           ──── або ────                 │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │  Email                            │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │           Продовжити              │  │
│  └───────────────────────────────────┘  │
│                                         │
└─────────────────────────────────────────┘
```

## Поведінка

1. Юзер вводить email і натискає "Продовжити"
2. Показується прелоадер
3. Backend перевіряє: чи існує юзер і чи має пароль
4. В залежності від результату — один з трьох сценаріїв (див. [Сценарії email-авторизації](./03-email-scenarios.md))

> **Безпека:** Endpoint `check-email` захищається rate limit по зв'язці IP + email (не лише email), щоб запобігти DoS-блокуванню легітимних юзерів (див. [Захист від brute force](./11-brute-force.md)).
