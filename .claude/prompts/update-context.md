Виконай повний аудит кодової бази проєкту та онови файл CLAUDE.md.

## Інструкції

### Крок 1 — Аналіз

Рекурсивно переглянь ВСЮ кодову базу проєкту (кожен файл, кожну директорію). Не пропускай нічого. Зрозумій:

- Загальну архітектуру та як модулі/компоненти пов'язані між собою
- Ключові бізнес-сутності та доменну модель
- Патерни, що повторюються (як організовані сервіси, контролери, утиліти)
- Залежності між модулями (хто кого імпортує, хто від кого залежить)
- Схему БД (Prisma/TypeORM/Drizzle або інше) та зв'язки між моделями
- API ендпоінти та їх структуру
- Middleware, guards, interceptors, pipes — що є і де
- Конфігурацію (env, config files) — які змінні критичні
- Тести — де є, яка стратегія тестування

### Крок 2 — Збережи існуючі правила

Прочитай поточний CLAUDE.md. Весь контент між маркерами <!-- MANUAL:START --> та <!-- MANUAL:END --> збережи в оновленому CLAUDE.md БЕЗ ЖОДНИХ ЗМІН — дослівно, включно з маркерами. Решту контенту можна повністю переписати на основі аналізу.

### Крок 3 — Запис

Перепиши/допиши CLAUDE.md з такою структурою:

# {Назва проєкту}

> Одне речення — що робить цей проєкт

## Tech Stack

Фреймворк, мова, ORM, основні бібліотеки з версіями

## Architecture Overview

Короткий опис архітектурного підходу (monolith/modular monolith/microservices, шари)

## Project Structure

Дерево директорій з коротким поясненням кожної ключової папки.
Не лістити кожен файл — тільки директорії та файли, які критично важливі.

## Domain Model

Ключові бізнес-сутності, їх зв'язки, enum'и.
Якщо є Prisma schema — коротко опиши моделі та relations.

## Module Dependency Map

Які модулі залежать від яких. Формат:

- ModuleA → залежить від ModuleB, ModuleC
- ModuleB → standalone

## Key Patterns

Патерни, що використовуються в проєкті. Для кожного — один конкретний приклад файлу:

- Як створюється новий endpoint
- Як організована валідація
- Як працює авторизація
- Як обробляються помилки
- Інші повторювані патерни

## API Overview

Згрупований список ендпоінтів по модулях (метод, шлях, короткий опис)

## Configuration & Environment

Критичні env змінні та що вони роблять

## Common Commands

Команди для запуску, тестування, міграцій, лінтингу, білду

## Rules & Conventions

⚠️ ЦЮ СЕКЦІЮ ЗБЕРЕЖИ З ПОТОЧНОГО CLAUDE.md БЕЗ ЗМІН.
Якщо її немає — створи базову з виявлених патернів.

## Known Complexities

Місця в коді, які неочевидні, мають workaround'и або потребують особливої уваги.
Нетривіальні архітектурні рішення з поясненням ЧОМУ так зроблено (якщо зрозуміло з коду).

### Вимоги до стилю

- Пиши лаконічно. Це довідник для AI-агента, не документація для людей.
- Конкретика > абстракція. Вказуй точні шляхи до файлів.
- Якщо щось неочевидно з коду — зазнач це явно як "[ПОТРЕБУЄ УТОЧНЕННЯ]"
- Мова файлу: українська для описів, англійська для технічних термінів та назв
